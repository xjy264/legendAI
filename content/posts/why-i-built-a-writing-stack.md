I do not want my technical writing to live in one-off tools.

The goal is a stack that behaves like a working notebook. I can post a paper note today, a project summary next week, and an older retrospective whenever I have time to reconstruct it. The site should absorb that uneven rhythm instead of fighting it.

That is why the archive keeps both Markdown and database rows. Markdown is the source of truth for the text itself. The database is only there to make the site feel fast and structured.

Long term, the real test is not whether the homepage looks good on day one. It is whether I still want to write here after a year.
